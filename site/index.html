Hermes Agent runtime evaluation

Benchmark the whole Hermes configuration, not just the model.

HermesBench measures whether a Hermes Agent setup reliably reaches useful, truthful, stable conclusions across daily technical-user workflows. It is a public benchmark and a reusable scenario harness for single-turn, multi-turn, and opt-in multi-profile checks.

Execution surfaces

Same use cases, visible runtime shape

HermesBench keeps one leaderboard, but it exposes whether a run used direct front-desk execution or kanban delegation. Use cases should remain compatible with both surfaces; kanban-enabled profiles may score better on delegation, progress, and long-work closure behavior.

Direct/no-kanban

Front-desk profile handles the prompt surface without kanban routing.

  • simpler profile surface
  • no worker profile publication required unless a runtime suite uses workers
  • baseline for non-kanban Hermes setups

Kanban delegation

Kanban toolset/config/routing is available for delegation-oriented behavior.

  • larger shared configuration surface
  • worker profiles must be disclosed when exercised
  • opt-in delegated_closure checks the multi-profile path

Reusable harness

Bring your own local suites

Public suites make scores comparable. Local suites make the framework useful for your own Hermes deployment. Load private JSON or YAML suites without changing the runner, store, report, or dashboard code. Cases define prompts, fixtures, checks, and driver policy; run configuration chooses the target agent. Use initial_prompt for one turn or turns for a multi-turn conversation. 

hermesbench --suite-path ./my-suites --list-suites
HERMES_RUN_LLM_EVALS=1 hermesbench \
  --suite-path ./my-suites \
  --suite team_ops_status

Distribution-aware baselines

Share the agent shape behind the score

HermesBench results are most useful when tied to a Hermes profile distribution: a git repo containing the agent's distribution.yaml, SOUL, config, skills, cron, and MCP wiring. Credentials, memories, sessions, state, logs, and local files stay private. If a result exercises delegated or multi-worker execution, every involved worker profile must be published or redacted as part of the baseline. 

hermes profile install github.com/you/my-agent --alias
HERMES_RUN_LLM_EVALS=1 hermesbench --high-rate --trials 1

# Publish score.json plus distribution repo or redacted baseline.

Install and run

Quick start

pip install git+https://github.com/verkyyi/hermesbench.git

hermesbench --list-suites
hermesbench --validate
HERMES_RUN_LLM_EVALS=1 hermesbench
HERMES_RUN_LLM_EVALS=1 hermesbench --high-rate --trials 1
HERMES_RUN_LLM_EVALS=1 hermesbench --json --no-store

Scoring

Reliability before capability

Closure Artifact correctness Stability Scope discipline Responsiveness Appropriateness Coherence

HermesBench scores deterministic evidence first: terminal outcome, artifact checks, stability, scoped side effects, and latency. The LLM judge is bounded to semantic appropriateness and coherence. Hard external-agent capability benchmarks remain useful, but HermesBench asks whether this runtime configuration is reliable enough to use every day.
